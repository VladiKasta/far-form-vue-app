# 📋 Quiz Form (Vue 3)

Многошаговая форма (quiz) на Vue 3 с валидацией и динамическими вопросами.

---

## 🚀 Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <your-repo-url>
cd <project-folder>
```

---

### 2. Установить зависимости

```bash
npm install
```

или

```bash
yarn install
```

---

### 3. Запустить проект

```bash
npm run dev
```

или

```bash
yarn dev
```

---

### 4. Открыть в браузере

По умолчанию:

```
http://localhost:5173
```

---

## 🏗️ Сборка проекта

Для production-сборки:

```bash
npm run build
```

---

## 🔍 Превью production-сборки

```bash
npm run preview
```

---

## 📁 Структура проекта

```
src/
 ├── components/
 │    ├── questions/       # шаги (вопросы)
 │    ├── UI/              # UI-компоненты (иконки и т.д.)
 │    ├── Policy.vue       # чекбокс соглашения
 │
 ├── App.vue               # основной компонент (форма)
 ├── main.js               # точка входа
```

---

## ⚙️ Как работает

- Все ответы хранятся в одном объекте `answers`
- Каждый шаг — отдельный компонент
- Валидация задаётся в конфиге `steps`
- Кнопка "Далее" активна только при валидных данных

---

## 🧠 Особенности

- Динамическая валидация (зависит от ответа пользователя)
- Поддержка файлов (upload + preview)
- Простая архитектура без дублирования состояния
- Масштабируемая структура шагов

---

## ✏️ Добавление нового шага

1. Создать компонент в `components/questions`
2. Добавить его в массив:

```js
const questionsComponents = [
  FirstQuestion,
  ...
]
```

3. Добавить конфиг в `steps`:

```js
{
  id: X,
  label: 'Ваш вопрос',
  required: ['fieldName']
}
```

## 🧪 Рекомендации

- Использовать один источник данных (`answers`)

- Валидацию держать в конфигурации, а не в компонентах
